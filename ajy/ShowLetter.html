Subject: Re: NetSim
 

Hi Richard

I hope you are back from your holiday (I'll be worried if you're not!)

I've been using NetSim, and have enjoyed it. There is some good 
programming 
there, and I am sure it will be useful for its intended purpose.

I've collected a series of comments.

(1) I am using it in Windows at 800*600, and there are problems. The 
toolbar at the bottom is not readable (some text is not visible) and 
the 
component toolbar at the top has a scrollbar (it only scrolls one 
pixel!) 
See enclosed JPEG (the lossy nature makes the text look a bit crap, but 
you 
get the idea)

(2) Would be nice to have a Zoom-to-fit icon in the top left toolbar. 
Zoom-to-fit should also shrink the workspace to remove any whitespace 
(see 
enclosed JPEG, which is "zoom-to-fit"ed)

(3) If you add a cable and then change your mind, you can't delete it. 
Additionally, it would be nice to be able to drag an entire 
*unattached* 
cable around by clicking on the middle of the cable (once one end is 
plugged in then this shouldn't be doable any more)

(4) Hard to do I know, but I found it incredibly annoying that a new 
component being added always appears top-left rather than simply 
dragging 
it off the component toolbar and dropping it on the workspace. This is 
doubly annoying when the screen is scrolled, as the component could 
easily 
appear off the visible part of the screen. I am sure this can be fixed 
if 
some simplifying assumptions are made (e.g. you can't scroll the 
workspace 
while dragging to add a component)

(5) When you type in an IP address for traceroute and ping, it would be 
nice to have a drop-down box of recently used IP addresses (analogous 
to  the BASH command history I suppose)

(6) When you do "list routing table" it should include the "local" 
routes 
done automatically when an IP address is added

(7) In "add explicit and default route" and "list routing table", can 
you 
remove the "ethernet card" option. I know it's correct, but I want to 
keep 
things simple

(8) When you "list IP addresses" can it be more like the output of 
"ifconfig" (i.e. card#/IP addr/netmask/network number/MTU/...)

(9) I've got Ping to work even when the hub is turned off. Would it be 
possible to have a hub icon with a red "power light" on it (no problem 
if 
it's not easy). To extend this to its logical conclusion, you could 
also 
have a green "transmit" light that blinks when a ping or a traceroute 
uses 
a hub (again, this is just decorative, but would be quite nice)

(10) If you have a dangling cable from a hub then you get an error when 
you 
try to ping through that hub

(11) If I do download or upload and then click "cancel" it says 
"invalid 
URL" rather than just canceling

(12) I did a traceroute with two networks and one gateway (enclosed) 
and it 
gave me 4 things in the output (it should be 2)

(13) "test connectivity" on the enclosed network gave me a score of 50% 
but 
didn't give me any IP addresses of things that couldn't communicate.

Enough to be doing on with here!

I hope to see you tomorrow
